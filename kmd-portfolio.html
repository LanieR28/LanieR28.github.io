Selected Works

Projects that connect data, systems, and visual experience.

01 / Main Project

CHUNITHM Visual Bot System

A personal system that connects social-media bot interaction, data synchronization, search assistance, score registration, and image-based visual output for a rhythm game community. The project emphasizes how raw game records can be reorganized into readable cards, rankings, progress views, and decision-support visuals.

  • Designed multiple visual formats for the same score data: Best 30, AJ, AJC, and PC views.
  • Built user-facing commands for search, registration, help, and synchronized data updates.
  • Focused on visibility, comparison, and fast comprehension in a social-media context.
View project detail

02 / Information Design

Song Information Card

A compact card interface that gathers title, difficulty, chart information, and visual cues. The aim is to reduce lookup friction and make information returned by a bot immediately usable.

03 / Data-to-Decision

Border Analysis View

A visual calculation tool that converts numerical score margins into an easier-to-understand representation of remaining tolerance and target achievement conditions.

04 / Progress Visualization

Level and User Progress Views

Two perspectives on the same dataset: an overall chart list and a personalized progress view. The design separates system-wide browsing from individual decision making.

05 / Web Tool

Game Resource Calculator

A lightweight web-based calculator for planning future in-game resources. It reflects an interest in building small, practical tools that support personal decision making through clear interfaces.

Relevance to KMD

Why these works matter as accomplishments.

Prototype-oriented thinking

The works are not only static images. They are connected to actual commands, data processing, rendering workflows, and usage scenarios, showing an interest in making ideas testable.

Media and communication

The projects examine how information is returned through media formats such as bot messages, visual cards, web pages, and compact interaction flows.

Evaluation-aware design

The design focus is not only appearance, but whether users can compare, understand, and act on system outputs more easily.
