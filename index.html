Project Summary

CHUNITHM Bot System

QQ 上で動作する Bot を基盤に、Best 30、楽曲情報、譜面プレビュー、レベル別一覧、 許容値分析、アカウント管理、公式サイトからのデータ同期までを扱う個人制作システムです。

  • Best 30 / AJ / AJC / PC30 の画像化表示
  • 楽曲情報カードと譜面プレビュー画像の生成
  • 別名検索、難易度補完、手動登録、プレイ回数登録
  • CHUNITHM-NET からのデータ取得と統合

Overview

研究・制作の狙い

既存の音楽ゲーム補助ツールは、情報取得、画像表示、ユーザー管理、同期処理が分断されていることが多く、 使い手にとっても説明する側にとっても全体像が見えにくいという課題があります。 本システムでは、Bot 操作、描画、データ処理、同期基盤を一つの流れとして統合し、 「実際に使えること」と「外部に見せられること」の両方を重視しています。

対象 CHUNITHM プレイヤー向け支援システム
主な要素 Bot / Rendering / Data Pipeline / Web Demo
重視点 視覚的な分かりやすさ・実用性・デモ適性

System Architecture

構成要素

  • Bot 層：QQ 公式 Bot と NapCat の両方を保持し、運用経路を分離
  • 命令処理層：コマンド解析、難易度補完、別名解決、アカウント処理
  • 描画層：B30、楽曲情報、譜面、レベル一覧、許容値分析を個別レンダラで生成
  • データ層：楽曲データ、プレイヤー成績、別名辞書、バインド情報を統合管理
  • 同期層：CHUNITHM-NET からの取得結果を解析し、内部データへ反映
